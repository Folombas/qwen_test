## 🎯 Приоритет 1: Критичные улучшения

### 🔐 Регистрация и авторизация (Go78)

**Проблема:** Сейчас все данные хранятся в localStorage, нет синхронизации между устройствами.

**Решение:**
```
┌─────────────────────────────────────────────┐
│           Авторизация                      │
│  ┌─────────┐  ┌─────────┐  ┌────────────┐ │
│  │  Email  │  │  OAuth  │  │  Telegram  │ │
│  │/Password│  │ (Google)│  │   Bot      │ │
│  └────┬────┘  └────┬────┘  └─────┬──────┘ │
│       │            │              │        │
│  ┌────▼────────────▼──────────────▼──────┐ │
│  │         JWT Tokens                    │ │
│  │    (Access + Refresh)                 │ │
│  └────┬──────────────────────────────────┘ │
│       │                                    │
│  ┌────▼──────────────────────────────────┐ │
│  │      PostgreSQL + Redis               │ │
│  │  - users (id, email, hash, created)   │ │
│  │  - sessions (user_id, token, exp)     │ │
│  │  - profiles (user_id, stats, settings)│ │
│  └───────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

**Функционал:**
- [ ] Регистрация (email + пароль)
- [ ] Вход/выход
- [ ] JWT токены (access + refresh)
- [ ] OAuth (Google, GitHub)
- [ ] Восстановление пароля
- [ ] Подтверждение email
- [ ] Смена пароля
- [ ] 2FA (опционально)

**Файлы:**
```
internal/
├── auth/
│   ├── jwt.go           # JWT генерация/валидация
│   ├── middleware.go    # Auth middleware
│   ├── handlers.go      # HTTP handlers
│   └── service.go       # Бизнес-логика
├── models/
│   └── user.go          # User модель
└── database/
    └── users.go         # Репозиторий пользователей
```

**API:**
```
POST   /api/auth/register     # Регистрация
POST   /api/auth/login        # Вход
POST   /api/auth/logout       # Выход
POST   /api/auth/refresh      # Refresh токена
POST   /api/auth/forgot       # Восстановление
GET    /api/auth/verify/:token # Подтверждение email
GET    /api/auth/me           # Текущий пользователь
PUT    /api/auth/me           # Обновление профиля
```

## 🎯 Приоритет 3: Инфраструктура

### 🐳 Docker и CI/CD (Go86)

**Docker:**
```dockerfile
FROM golang:1.21-alpine AS builder
# Build Go binary

FROM nginx:alpine
# Serve static + proxy to Go
```

**docker-compose.yml:**
```yaml
version: '3.8'
services:
  app:
    build: .
    ports:
      - "8080:80"
    depends_on:
      - postgres
      - redis
  
  postgres:
    image: postgres:15
    volumes:
      - pgdata:/var/lib/postgresql/data
  
  redis:
    image: redis:7-alpine
  
  adminer:
    image: adminer
    ports:
      - "8081:8080"
```

**GitHub Actions:**
```yaml
name: CI/CD
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-go@v4
      - run: go test ./...
  
  build:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - build Docker image
      - push to registry
  
  deploy:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - deploy to server
```
